  1. Training final model over DBpedia is going to be a computationally expensive task. Any addition of entities will cause the model to be modified accordingly.  (Possible answer is the name-tag based method).
  1. The essence of NsPM is to create a single model to take care of all the aspects of conversion of  natural language to SPARQL.
  2. Traditional approaches employ different modules for different aspects of Natural Language to SPARQL like entity detection, Syntax check etc.. Each module contributes to the final error in SPARQL query prediction model.
  3. This single model helps in preventing error that propagates through different modules as used by many other traditional mechanisms.
